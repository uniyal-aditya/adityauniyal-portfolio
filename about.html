My obsession with how things actually work — not just that they work — started early. What makes a bot respond in milliseconds? What separates code you can extend from code that breaks when you breathe on it?

// How I work

My principles.

01

System before syntax

Design the architecture. Map the data flows. Think through edge cases. Then open the editor.

02

Readable over clever

The next developer — including future me — deserves code they can understand and extend without dread.

03

Performance is a feature

Every wasted millisecond compounds. I optimize early and deliberately, not as an afterthought.

04

Ship, then refine

Shipped software that works beats unfinished perfection every time. Iterate in the open.

05

UI earns logic's trust

Elegant interfaces and robust logic are not in conflict. Great software demands both.

06

Always learning

Technology evolves. Intellectual curiosity and depth of study are not optional — they're the job.
